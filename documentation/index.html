How to Request Topics for Grand Magus Alistair's Chronicles

üìú What is This?

The Blog Request System allows you to suggest topics, events, or themes for Grand Magus Alistair to cover in his daily chronicles at midwestmage.blog.

How it works: You submit requests through the admin panel, and Claude AI considers them when generating each day's blog post at 12 PM Central Time. Your requests stay active until used or manually removed.

üéØ How to Submit Requests

1Access the Admin Panel

Visit admin.midwestmage.blog

Note: Access is restricted via Cloudflare Access (email authentication)

2Fill Out the Request Form

  • Request Text: Describe what you want Alistair to cover
  • Duration: How long should this request stay active?
  • Priority: How urgent is this request?
  • Expiry Date: Optional - automatically expire on a specific date

3Submit and Wait

After submitting, your request will appear in the "Manage Requests" tab within 30-60 seconds.

üè∑Ô∏è Request Tags

Your requests are automatically formatted with tags based on your selections:

[PRIORITY:HIGH] [PRIORITY:NORMAL] [PRIORITY:LOW]
[ONE-TIME] [TODAY-ONLY] [THIS-WEEK] [RECURRING]
[EXPIRES:2025-10-31]

Examples:

- [ ] [PRIORITY:HIGH] [TODAY-ONLY] Cover my successful 8-point buck harvest near Sioux Falls
- [ ] [THIS-WEEK] Talk about the foggy weather we've been having
- [ ] [RECURRING] Make more jokes about cryptocurrency as "magical tokens"
- [ ] [EXPIRES:2025-10-31] Mention the Rosebud elk hunt deadline

‚è±Ô∏è Timeline

Understanding how long things take:

Instant: You submit request via admin panel
~30-60 seconds: Request appears in your "Manage Requests" tab
Daily at 12 PM Central: Blog post generated considering all active requests
~5-10 minutes later: New blog post published to midwestmage.blog

‚úèÔ∏è Managing Requests

View Active Requests

Go to the "Manage Requests" tab in the admin panel and click "Refresh Requests" to see all active topic requests.

Remove Requests

Click the red "Remove" button next to any request. A themed confirmation modal will appear. Once confirmed, the request is permanently removed within seconds.

Direct File Access

Advanced users can view or edit requests directly on GitHub:

  • View: BLOG_REQUESTS.md
  • All requests are stored in a public repository for transparency

üí° Tips for Good Requests

Specific Events

  • Include dates: "Cover my pheasant hunt on October 20th"
  • Include locations: "Mention my successful harvest near McCook County"
  • Include details: "Two hero dogs guarding turkeys on public land"

Recurring Themes

  • "Focus on outdoor recreation"

üîí Security & Privacy

  • Admin Panel: Protected by Cloudflare Access (email whitelist only)
  • Requests Repository: Publicly viewable on GitHub (no sensitive info stored)
  • API: CORS-protected Cloudflare Worker with encrypted GitHub token
  • Authentication: Email PIN or OAuth-based access

üõ†Ô∏è Technical Details

For those interested in how it works behind the scenes:

Architecture:
  • Frontend: Cloudflare Pages (admin.midwestmage.blog)
  • Backend: Cloudflare Worker (serverless API)
  • Storage: GitHub repository (mage-blog-requests)
  • Automation: GitHub Actions workflows
  • AI: Claude API for content generation

The entire system is built on serverless architecture with no traditional backend servers. Requests are stored as markdown in a Git repository, providing full version history and transparency.

‚ùì Troubleshooting

Request not appearing in admin panel?

  • Wait 60 seconds and click "Refresh Requests"
  • Check GitHub Actions to see if workflow ran
  • Verify you're logged into the admin panel

Request not used in blog post?

  • AI considers requests but doesn't guarantee usage
  • High priority requests are more likely to be included
  • Recurring requests will be considered in future posts
  • Very specific requests may take a few days to incorporate naturally

Can't access admin panel?

  • Verify your email is whitelisted in Cloudflare Access
  • Check for email PIN if using email authentication
  • Try clearing browser cache and cookies
